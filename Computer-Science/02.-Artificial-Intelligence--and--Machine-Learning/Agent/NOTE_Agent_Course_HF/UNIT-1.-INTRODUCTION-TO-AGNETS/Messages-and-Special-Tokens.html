1.3 Messages and Special Tokens

Jan 27, 20268 min read

In this section we are going to talk about how LLM structure their generations through chat templates.

An important thing is to notice that what we see in a chat UI like ChatGPT and what happen behind the UI are different. When your are typing with ChatGPT, your feel like that your are using a chat Messages, rather than a single prompt sequence. But this is in fact a UI abstraction. Before being fed into the LLM, all the messages in the conversation are concatenated and formatted into a single prompt that the LLM can understand. The model does not ‚Äúremember‚Äù the conversation: it reads it in full every time.

We can see here the difference between what we see in UI and the prompt fed to the model:

This is where the template come in. They act as the bridge between conversational messages (including user and assistant turns) and the specific formatting requirements of your chosen LLM.

What templates do?

chat templates structure the communication between the user and the agent, ensuring that every model‚Äîdespite its unique special tokens‚Äîreceives the correctly formatted prompt

We mentioned EOS¬†(End Of Sequence) token before, and now are talking about special tokens again. special tokens are what models use to delimit where the user and assistant turns start and end. Different models would also use different formatting rules and delimiters for the messages in the conversation.

Chat-Templates

We now know that Chat-Templates guide how message exchanges are formatted into a single prompt. Let‚Äôs go deeper in it.

Base Models vs. Instruct Models

  • A Base Model¬†is trained on raw text data to predict the next token.

  • An¬†Instruct Model¬†is fine-tuned specifically to follow instructions and engage in conversations.

For example,¬†SmolLM2-135M¬†is a base model, while¬†SmolLM2-135M-Instruct¬†is its instruction-tuned variant.

A Base Model is like a super-powerful autocomplete engine, it may be smart as well, but no preference for being helpful, safe, or following instructions. An Instruct Model are start from a base model, then be fine-tuned to follow instructions, answer questions and be helpful, safe, and conversational. This is usually done via supervised fine-tuning (human-written Q&A) and RLHF, which we are not going to expand here.

To make a Base Model behave like an instruct model (Base Model ‚Üí instruct model), we need to¬†format our prompts in a consistent way that the model can understand. This is where chat templates come in.

ChatML (Chat Makeup Language) is one such template format that structures conversations with clear role indicator (system, user, assistant). If you have interacted with some AI API lately, you know that is the standard practice.

A base model could be fine-tuned on different chat templates, so when we‚Äôre using an instruct model we need to make sure we‚Äôre using the correct chat template.

Understanding Chat Templates

Each instruct model uses different conversation formats and special tokens, so chat templates are implemented to ensure that we correctly format the prompt the way each model expects.

In¬†transformers, chat templates include¬†Jinja2 code¬†that describes how to transform the ChatML list of JSON messages, as presented in the above examples, into a textual representation of the system-level instructions, user messages and assistant responses that the model can understand.

This structure helps maintain consistency across interactions and ensures the model responds appropriately to different types of inputs.

Below is a simplified version of the¬†SmolLM2-135M-Instruct¬†chat template:

{% for message in messages %}
{% if loop.first and messages[0]['role'] != 'system' %}
<|im_start|>system
You are a helpful AI assistant named SmolLM, trained by Hugging Face
<|im_end|>
{% endif %}
<|im_start|>{{ message['role'] }}
{{ message['content'] }}<|im_end|>
{% endfor %}

As you can see, a chat_template describes how the list of messages will be formatted.
